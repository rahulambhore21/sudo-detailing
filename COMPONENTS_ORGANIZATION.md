# Components Organization\n\nThis document describes the organized component structure for the luxury car detailing website.\n\n## File Structure\n\n```\ncomponents/\n├── Homepage.tsx                 # Main homepage component (orchestrator)\n├── ContactForm.tsx              # Contact form modal component\n├── types/\n│   └── index.ts                 # Shared TypeScript types\n├── shared/                      # Reusable UI components\n│   ├── index.ts                 # Barrel export file\n│   ├── Arrow.tsx                # Arrow icon component\n│   ├── ArrowIconWrapper.tsx     # Arrow icon wrapper with styling\n│   └── PlayButtonArrowhead.tsx  # Play button arrow component\n├── sections/                    # Page section components\n│   ├── index.ts                 # Barrel export file\n│   ├── Navbar.tsx               # Navigation header\n│   ├── Hero.tsx                 # Hero section with main heading\n│   ├── Services.tsx             # Services grid section\n│   ├── Showreel.tsx             # Video showreel section\n│   ├── Features.tsx             # Features/benefits section\n│   ├── CTA.tsx                  # Call-to-action section\n│   └── Footer.tsx               # Footer section\n└── ui/                          # shadcn/ui components (unchanged)\n    └── ...\n```\n\n## Component Descriptions\n\n### Main Components\n\n- **Homepage.tsx**: The main page component that orchestrates all sections and handles GSAP animations\n- **ContactForm.tsx**: Modal component for the contact form\n\n### Types (`components/types/`)\n\n- **index.ts**: Contains shared TypeScript interfaces and types, including `SectionProps`\n\n### Shared Components (`components/shared/`)\n\nReusable UI components that can be used across multiple sections:\n\n- **Arrow.tsx**: Simple arrow icon for navigation and buttons\n- **ArrowIconWrapper.tsx**: Styled arrow icon wrapper with optional className prop\n- **PlayButtonArrowhead.tsx**: Play button arrow for video/showreel components\n- **index.ts**: Barrel export for clean imports\n\n### Section Components (`components/sections/`)\n\nEach major page section has its own component:\n\n- **Navbar.tsx**: Site navigation with logo and CTA button\n- **Hero.tsx**: Main hero section with title, subtitle, and hero image\n- **Services.tsx**: Grid of services offered with images and descriptions\n- **Showreel.tsx**: Video showcase section with play button\n- **Features.tsx**: \"Why Choose Us\" section with feature list\n- **CTA.tsx**: Call-to-action section with background image\n- **Footer.tsx**: Site footer with links and company info\n- **index.ts**: Barrel export for clean imports\n\n## Benefits of This Structure\n\n1. **Modularity**: Each component has a single responsibility\n2. **Reusability**: Shared components can be easily reused\n3. **Maintainability**: Easy to locate and update specific components\n4. **Clean Imports**: Barrel exports make imports cleaner\n5. **Type Safety**: Centralized types ensure consistency\n6. **Separation of Concerns**: UI components, business logic, and types are separated\n\n## Import Examples\n\n```tsx\n// In Homepage.tsx\nimport { Navbar, Hero, Services, Showreel, Features, CTA, Footer } from './sections';\nimport { ContactForm } from './ContactForm';\n\n// In any section component\nimport { Arrow, ArrowIconWrapper } from '../shared';\nimport { SectionProps } from '../types';\n```\n\n## Animation Classes\n\nThe GSAP animations target these CSS classes:\n- `.navbar`, `.hero-title`, `.hero-subtitle`, `.hero-btn`, `.hero-car`\n- `.services-header`, `.service-card`\n- `.showreel-container`\n- `.features-left`, `.feature-item`\n- `.cta-container`\n- `.footer-section`\n\nThese classes are preserved in the separated components to maintain animation functionality.\n\n## Images\n\nPlaceholder images from Unsplash are used instead of Figma imports for better development workflow. Replace these with actual assets when available:\n\n- Hero car image: `/assets/luxury-car-hero.jpg`\n- Service images: Various Unsplash car detailing images\n- Feature image: Car detail work image\n- CTA background: Luxury car image\n- Showreel thumbnail: Car showcase image\n\n## Future Enhancements\n\n1. Add prop interfaces for better type safety\n2. Create a constants file for reusable values\n3. Add unit tests for individual components\n4. Consider lazy loading for performance\n5.
